# 开发日志

## 2026-04-07

### Phase 1: 基础骨架与配置页 - 开发完成

#### 开发内容

##### 1. 项目结构搭建
创建了标准的分层架构：
- `models/` - 数据模型层
- `services/` - 服务层
- `ui/` - UI层
- `data/` - 数据存储目录

##### 2. 数据模型层实现
- **`prize_model.py`**: 实现了 `PrizeItem` 奖项模型
  - 包含奖项名称和权重
  - 提供 `to_dict()` 和 `from_dict()` 序列化方法
  - 实现 `validate()` 验证方法

- **`config_model.py`**: 实现了 `AppConfig` 应用配置模型
  - 包含麦克风索引、音量阈值、揭晓时间、回退时间、奖项列表
  - 提供序列化和反序列化方法
  - 实现完整的配置验证逻辑

##### 3. 服务层实现
- **`config_service.py`**: 实现了 `ConfigService` 配置管理服务
  - `load_config()`: 从 JSON 文件加载配置，不存在时返回默认配置
  - `save_config()`: 保存配置到文件，保存前进行验证
  - `get_default_config()`: 提供默认配置（4个示例奖项）
  - `get_microphone_list()`: 获取麦克风列表（Phase 1 为模拟数据）
  - 自动创建数据目录

##### 4. UI层实现
- **`config_window.py`**: 实现了 `ConfigWindow` 配置界面
  - **麦克风设置组**: 下拉框选择麦克风设备
  - **活动参数设置组**: 
    - 音量阈值设置（0.01-1.0）
    - 揭晓时长设置（1-60秒）
    - 回退时长设置（1-60秒）
  - **奖项设置组**:
    - 表格显示奖项列表（名称、权重、操作）
    - 支持直接在表格中编辑奖项名称
    - 使用 SpinBox 编辑权重（1-1000）
    - 添加奖项按钮
    - 删除奖项按钮（至少保留1个）
  - **底部按钮**:
    - 保存配置按钮：验证并保存配置
    - 开始活动按钮：验证配置并显示摘要（Phase 3 将跳转到活动界面）

##### 5. 应用主入口
- **`app.py`**: 应用主入口文件
  - 创建 QApplication
  - 设置 Fusion 样式
  - 创建并显示配置窗口

##### 6. 配置文件
- **`data/config.json`**: 默认配置文件
  - 包含示例奖项：谢谢惠顾(50)、小玩具1个(20)、休息5分钟(20)、贴纸1张(10)
  
- **`requirements.txt`**: Python 依赖清单
  - PySide6>=6.5.0

#### 实现的功能特性

1. **配置持久化**: 配置自动保存到 JSON 文件，下次启动自动加载
2. **数据验证**: 多层验证机制，确保配置合法性
3. **用户友好的UI**: 使用分组框、表格、SpinBox 等控件提供良好的交互体验
4. **错误处理**: 提供清晰的错误提示信息
5. **默认配置**: 首次运行或配置丢失时提供合理的默认值

## 2026-04-07

### Phase 2: 音频采集与阈值检测 - 开发完成

#### 开发内容

##### 1. 更新依赖
- **`requirements.txt`**: 添加音频处理依赖
  - sounddevice>=0.4.6 (音频采集)
  - numpy>=1.24.0 (音频数据处理)

##### 2. 音频服务实现
- **`services/audio_service.py`**: 实现 `AudioService` 音频采集服务类（约270行）
  - **设备管理**:
    - `get_device_list()`: 获取系统音频输入设备列表
    - `get_default_input_device()`: 获取默认输入设备
    - `set_device()`: 动态切换音频设备
  - **音频采集**:
    - 使用 sounddevice 进行实时音频流采集
    - 支持自定义采样率、块大小
    - 异步回调处理音频数据
  - **音量计算**:
    - `calculate_volume()`: 计算RMS音量值
    - 归一化到 0.0-1.0 范围
  - **平滑处理**:
    - `smooth_volume()`: 移动平均平滑
    - 可配置平滑窗口大小（默认5帧）
    - 避免音量抖动
  - **生命周期管理**:
    - `start()`: 启动音频采集
    - `stop()`: 停止音频采集
    - 支持音量回调函数
  - **设备测试**:
    - `test_device()`: 测试设备是否可用

##### 3. 配置服务更新
- **`services/config_service.py`**: 更新麦克风列表获取方法
  - 使用 `AudioService.get_device_list()` 获取真实设备
  - 替换Phase 1的模拟数据
  - 失败时返回默认设备（兜底策略）

##### 4. 配置界面增强
- **`ui/config_window.py`**: 添加实时音量监测功能（约140行新增）
  - **UI组件**:
    - 实时音量值标签
    - 音量进度条（QProgressBar）
    - 阈值指示器
    - 开始/停止测试按钮
  - **实时监测**:
    - QTimer定时器（50ms刷新）
    - 实时显示音量值和进度条
    - 根据阈值动态改变进度条颜色
      - 超过阈值：绿色
      - 低于阈值：橙色
  - **交互功能**:
    - `on_toggle_test()`: 切换测试状态
    - `start_monitoring()`: 启动音量监测
    - `stop_monitoring()`: 停止音量监测
    - `update_volume_display()`: 更新音量显示
    - `on_mic_changed()`: 麦克风设备切换
  - **生命周期**:
    - `closeEvent()`: 窗口关闭时自动停止监测

#### 实现的功能特性

1. **真实设备支持**: 使用sounddevice获取系统实际音频设备
2. **实时音量监测**: 50ms刷新率的实时音量显示
3. **音量平滑**: 移动平均算法减少抖动
4. **阈值可视化**: 实时对比音量与阈值
5. **设备热切换**: 监测过程中可切换设备
6. **自动资源管理**: 窗口关闭自动清理音频资源

#### 代码统计

- 新增代码行数: 约410行
- 主要模块:
  - `audio_service.py`: ~270行
  - `config_window.py`: ~140行新增
  - 其他更新: ~10行
- 符合Phase 2目标（1000-1800行的下限）

#### 技术实现

1. **音频采集**: 
   - 使用sounddevice的InputStream进行非阻塞采集
   - 回调函数异步处理音频数据
   - 支持单声道输入

2. **音量计算**:
   - RMS (均方根) 算法
   - 归一化映射到0-1范围

3. **平滑算法**:
   - 固定窗口大小的移动平均
   - 使用deque高效实现
   - 默认5帧平滑窗口

4. **UI更新**:
   - QTimer定时器驱动
   - 避免在音频回调中直接更新UI
   - 线程安全的设计
