# 开发日志

## 2026-04-07

### Phase 1: 基础骨架与配置页 - 开发完成

#### 开发内容

##### 1. 项目结构搭建
创建了标准的分层架构：
- `models/` - 数据模型层
- `services/` - 服务层
- `ui/` - UI层
- `data/` - 数据存储目录

##### 2. 数据模型层实现
- **`prize_model.py`**: 实现了 `PrizeItem` 奖项模型
  - 包含奖项名称和权重
  - 提供 `to_dict()` 和 `from_dict()` 序列化方法
  - 实现 `validate()` 验证方法

- **`config_model.py`**: 实现了 `AppConfig` 应用配置模型
  - 包含麦克风索引、音量阈值、揭晓时间、回退时间、奖项列表
  - 提供序列化和反序列化方法
  - 实现完整的配置验证逻辑

##### 3. 服务层实现
- **`config_service.py`**: 实现了 `ConfigService` 配置管理服务
  - `load_config()`: 从 JSON 文件加载配置，不存在时返回默认配置
  - `save_config()`: 保存配置到文件，保存前进行验证
  - `get_default_config()`: 提供默认配置（4个示例奖项）
  - `get_microphone_list()`: 获取麦克风列表（Phase 1 为模拟数据）
  - 自动创建数据目录

##### 4. UI层实现
- **`config_window.py`**: 实现了 `ConfigWindow` 配置界面
  - **麦克风设置组**: 下拉框选择麦克风设备
  - **活动参数设置组**: 
    - 音量阈值设置（0.01-1.0）
    - 揭晓时长设置（1-60秒）
    - 回退时长设置（1-60秒）
  - **奖项设置组**:
    - 表格显示奖项列表（名称、权重、操作）
    - 支持直接在表格中编辑奖项名称
    - 使用 SpinBox 编辑权重（1-1000）
    - 添加奖项按钮
    - 删除奖项按钮（至少保留1个）
  - **底部按钮**:
    - 保存配置按钮：验证并保存配置
    - 开始活动按钮：验证配置并显示摘要（Phase 3 将跳转到活动界面）

##### 5. 应用主入口
- **`app.py`**: 应用主入口文件
  - 创建 QApplication
  - 设置 Fusion 样式
  - 创建并显示配置窗口

##### 6. 配置文件
- **`data/config.json`**: 默认配置文件
  - 包含示例奖项：谢谢惠顾(50)、小玩具1个(20)、休息5分钟(20)、贴纸1张(10)
  
- **`requirements.txt`**: Python 依赖清单
  - PySide6>=6.5.0

#### 实现的功能特性

1. **配置持久化**: 配置自动保存到 JSON 文件，下次启动自动加载
2. **数据验证**: 多层验证机制，确保配置合法性
3. **用户友好的UI**: 使用分组框、表格、SpinBox 等控件提供良好的交互体验
4. **错误处理**: 提供清晰的错误提示信息
5. **默认配置**: 首次运行或配置丢失时提供合理的默认值

## 2026-04-07

### Phase 3: 二层活动页与抽奖逻辑 - 开发完成

#### 开发内容

##### 1. 抽奖服务完善
- **`services/lottery_service.py`**: 完成抽奖服务实现（约93行）
  - **加权随机抽奖**:
    - `draw()`: 根据权重进行随机抽奖
    - 累积权重算法保证按概率分配
    - 兜底机制防止边界情况
  - **状态管理**:
    - `current_result`: 存储当前抽奖结果
    - `is_locked`: 标记结果是否已锁定
    - `lock_result()`: 锁定结果（揭晓完成后）
    - `reset()`: 重置状态，准备下一轮
  - **配置更新**:
    - `update_config()`: 支持动态更新配置
    - 保持当前抽奖状态不变

##### 2. 活动界面实现
- **`ui/activity_window.py`**: 实现活动界面窗口（约550行）
  - **UI布局**:
    - 标题与提示区域
    - 中央抽奖结果显示区（大号文字）
    - 音量监测区（进度条+数值）
    - 揭晓进度区（进度条+百分比）
    - 状态提示标签
    - 底部操作按钮（返回配置、下一轮）
  
  - **核心功能**:
    - `start_new_round()`: 开始新一轮抽奖
      - 重置状态变量
      - 预抽奖（提前确定结果）
      - 启动音频采集
      - 启动定时器
    - `update_display()`: UI刷新（50ms）
      - 更新音量条显示
      - 更新进度条显示
      - 根据阈值改变颜色
    - `update_progress()`: 进度逻辑更新（100ms）
      - 音量超阈值：进度前进
      - 音量低阈值：进度回退
      - 达到100%：触发揭晓完成
    - `update_result_display()`: 结果显示更新
      - 进度<30%: 显示"???"
      - 进度30-100%: 显示奖项名称（透明度变化）
      - 完全揭晓: 高亮显示（黄色背景+边框）
    - `on_fully_revealed()`: 揭晓完成处理
      - 锁定结果
      - 更新状态提示
      - 启用"下一轮"按钮
    - `on_reset()`: 开始下一轮
    - `on_back_to_config()`: 返回配置页
    - `stop_activity()`: 停止所有定时器和音频采集
  
  - **揭晓逻辑**:
    - **前进速度**: 根据 `reveal_seconds` 计算
      - 每100ms进度增量 = 0.1 / reveal_seconds
    - **回退速度**: 根据 `decay_seconds` 计算
      - 每100ms进度减量 = 0.1 / decay_seconds
    - **进度范围**: 0.0 - 1.0（对应0% - 100%）
    - **阈值判断**: 实时对比音量与配置阈值
  
  - **视觉效果**:
    - 音量条颜色：绿色（超阈值）/橙色（低阈值）
    - 结果透明度：随进度逐渐清晰
    - 完全揭晓：黄色高亮背景
    - 状态提示：蓝色（进行中）/绿色（完成）

##### 3. 应用控制器实现

    - 活动页 -> 配置页：关闭活动窗口，显示配置窗口

##### 4. 配置窗口更新

#### 技术实现

1. **定时器机制**:
   - UI 刷新定时器: 50ms（高频，保证流畅）
   - 进度更新定时器: 100ms（逻辑更新）
   - 分离显示和逻辑，避免冲突

2. **进度计算**:
   - 增量公式: Δt / target_time
   - Δt = 0.1秒（定时器间隔）
   - target_time = reveal_seconds 或 decay_seconds

3. **状态管理**:
   - `reveal_progress`: 0.0 - 1.0
   - `is_revealed`: 布尔标志
   - `current_result`: 当前奖项对象

4. **信号槽通信**:
   - `start_clicked` 信号：配置页 -> 控制器
   - `back_to_config` 信号：活动页 -> 控制器
   - 传递 AppConfig 对象

5. **资源管理**:

   - 定时器停止
   - 音频服务停止
